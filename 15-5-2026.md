1. Uber: Toll Plaza Revenue CalculationProblem: Calculate the total toll collected per day based on dynamic pricing. If a vehicle returns within 4 hours, they pay a discounted rate.Logic & RulesStandard Rates: Motorcycle (Free), Car (₹40), Bus (₹70), Truck (₹80).Return Rate (within 4 hours): Car (₹20), Bus (₹30), Truck (₹40).Key Insight: Use LAG() window functions to find the previous crossing time for each vehicle to calculate the duration between trips.SQL SolutionSQLWITH base AS (
    SELECT 
        vehicle_no, vehicle_type, crossing_time,
        DATE(crossing_time) AS crossing_date,
        LAG(crossing_time) OVER (PARTITION BY vehicle_no ORDER BY crossing_time) AS prev_crossing
    FROM toll_log
),
toll_calc AS (
    SELECT *,
        CASE 
            WHEN vehicle_type = 'motorcycle' THEN 0
            WHEN prev_crossing IS NOT NULL AND TIMESTAMPDIFF(HOUR, prev_crossing, crossing_time) <= 4
            THEN CASE 
                WHEN vehicle_type = 'car' THEN 20
                WHEN vehicle_type = 'bus' THEN 30
                WHEN vehicle_type = 'truck' THEN 40
            END
            ELSE CASE 
                WHEN vehicle_type = 'car' THEN 40
                WHEN vehicle_type = 'bus' THEN 70
                WHEN vehicle_type = 'truck' THEN 80
            END
        END AS toll_amount
    FROM base
)
SELECT crossing_date, SUM(toll_amount) AS total_toll
FROM toll_calc
GROUP BY crossing_date;
PySpark SolutionPythonfrom pyspark.sql import functions as F
from pyspark.sql.window import Window

w = Window.partitionBy("vehicle_no").orderBy("crossing_time")

df_calc = df.withColumn("prev_time", F.lag("crossing_time").over(w)) \
    .withColumn("hours_diff", (F.col("crossing_time").cast("long") - F.col("prev_time").cast("long")) / 3600)

df_final = df_calc.withColumn("toll_amount", 
    F.when(F.col("vehicle_type") == "motorcycle", 0)
    .when((F.col("hours_diff") <= 4) & F.col("prev_time").isNotNull(),
        F.when(F.col("vehicle_type") == "car", 20)
         .when(F.col("vehicle_type") == "bus", 30)
         .otherwise(40)) # Truck
    .otherwise(
        F.when(F.col("vehicle_type") == "car", 40)
         .when(F.col("vehicle_type") == "bus", 70)
         .otherwise(80)) # Truck
)

df_final.groupBy(F.to_date("crossing_time").alias("date")).agg(F.sum("toll_amount")).show()

Walmart: Find Palindrome KeysProblem: Identify keys where the sequence of associated values reads the same forward and backward.Logic & RulesStep 1: Aggregate values into a list/array for each key while maintaining order.Step 2: Compare the array to its reversed version.Why? This avoids expensive self-joins and recursive logic.SQL SolutionSQLWITH ordered_data AS (
    SELECT key, value, 
           ROW_NUMBER() OVER (PARTITION BY key ORDER BY key) AS rn 
    FROM input_table
),
grouped_data AS (
    SELECT key, ARRAY_AGG(value ORDER BY rn) AS seq_values
    FROM ordered_data
    GROUP BY key
)
SELECT key FROM grouped_data
WHERE seq_values = REVERSE(seq_values);
PySpark SolutionPythonfrom pyspark.sql import functions as F
from pyspark.sql.window import Window

window_spec = Window.partitionBy("key").orderBy("key")
df_ordered = df.withColumn("rn", F.row_number().over(window_spec))

df_grouped = df_ordered.groupBy("key").agg(F.collect_list("value").alias("seq"))

result = df_grouped.filter(F.col("seq") == F.reverse(F.col("seq"))).select("key")
result.show()

Amazon: Valid Triangle DetectionProblem: Given a table where each triangle side is a separate row, determine which combinations form a valid triangle.Logic & RulesTriangle Inequality Theorem: A triangle is valid if $A+B > C$, $A+C > B$, and $B+C > A$.Strategy: Pivot the data so that sides A, B, and C for a single ID are in the same row.SQL SolutionSQLSELECT CombinationID
FROM triangles
GROUP BY CombinationID
HAVING 
    MAX(CASE WHEN SideID = 'A' THEN SideLength END) + MAX(CASE WHEN SideID = 'B' THEN SideLength END) > MAX(CASE WHEN SideID = 'C' THEN SideLength END)
    AND MAX(CASE WHEN SideID = 'A' THEN SideLength END) + MAX(CASE WHEN SideID = 'C' THEN SideLength END) > MAX(CASE WHEN SideID = 'B' THEN SideLength END)
    AND MAX(CASE WHEN SideID = 'B' THEN SideLength END) + MAX(CASE WHEN SideID = 'C' THEN SideLength END) > MAX(CASE WHEN SideID = 'A' THEN SideLength END);
PySpark SolutionPythonfrom pyspark.sql import functions as F

# Pivot rows to columns
pivot_df = df.groupBy("CombinationID").pivot("SideID", ["A", "B", "C"]).agg(F.max("SideLength"))

# Filter based on triangle logic
result = pivot_df.filter(
    (F.col("A") + F.col("B") > F.col("C")) &
    (F.col("A") + F.col("C") > F.col("B")) &
    (F.col("B") + F.col("C") > F.col("A"))
).select("CombinationID")

result.show()
