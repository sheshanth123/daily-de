### Walmart Data Engineer II Interview Question

**The Challenge:**
Find customers whose cumulative revenue crossed **$10,000** for the first time **last year**.

**Requirements:**
* Only include **completed orders**.
* **Ignore** refund transactions.
* Count the customer **only in the month** they first cross the threshold.

---

### SQL Solution

```sql
-- Step 1: Filter for valid, completed orders
WITH valid_orders AS (
    SELECT 
        order_id,
        customer_id,
        order_date,
        order_amount
    FROM orders
    WHERE order_status = 'COMPLETED'
),

-- Step 2: Compute running total per customer using Window Functions
cumulative_revenue AS (
    SELECT
        customer_id,
        order_date,
        SUM(order_amount) OVER (
            PARTITION BY customer_id
            ORDER BY order_date
            ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
        ) AS cum_revenue
    FROM valid_orders
),

-- Step 3: Identify the previous revenue state to detect the "crossing" moment
threshold_crossing AS (
    SELECT *,
           LAG(cum_revenue) OVER (
               PARTITION BY customer_id
               ORDER BY order_date
           ) AS prev_cum_revenue
    FROM cumulative_revenue
),

-- Step 4: Filter for the specific event where revenue hits >= 10,000 for the first time
first_crossing AS (
    SELECT
        customer_id,
        order_date
    FROM threshold_crossing
    WHERE cum_revenue >= 10000 
      AND (prev_cum_revenue IS NULL OR prev_cum_revenue < 10000)
)

-- Step 5: Filter for the previous calendar year and group by month
SELECT
    customer_id,
    DATE_TRUNC('month', order_date) AS crossing_month
FROM first_crossing
WHERE EXTRACT(YEAR FROM order_date) = EXTRACT(YEAR FROM CURRENT_DATE) - 1;
