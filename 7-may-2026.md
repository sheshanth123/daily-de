### Walmart Data Engineer II Interview Question

**The Challenge:**
Find customers whose cumulative revenue crossed **$10,000** for the first time **last year**.

**Requirements:**
* Only include **completed orders**.
* **Ignore** refund transactions.
* Count the customer **only in the month** they first cross the threshold.

---

### SQL Solution

```sql
-- Step 1: Filter for valid, completed orders
WITH valid_orders AS (
    SELECT 
        order_id,
        customer_id,
        order_date,
        order_amount
    FROM orders
    WHERE order_status = 'COMPLETED'
),

-- Step 2: Compute running total per customer using Window Functions
cumulative_revenue AS (
    SELECT
        customer_id,
        order_date,
        SUM(order_amount) OVER (
            PARTITION BY customer_id
            ORDER BY order_date
            ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
        ) AS cum_revenue
    FROM valid_orders
),

-- Step 3: Identify the previous revenue state to detect the "crossing" moment
threshold_crossing AS (
    SELECT *,
           LAG(cum_revenue) OVER (
               PARTITION BY customer_id
               ORDER BY order_date
           ) AS prev_cum_revenue
    FROM cumulative_revenue
),

-- Step 4: Filter for the specific event where revenue hits >= 10,000 for the first time
first_crossing AS (
    SELECT
        customer_id,
        order_date
    FROM threshold_crossing
    WHERE cum_revenue >= 10000 
      AND (prev_cum_revenue IS NULL OR prev_cum_revenue < 10000)
)

-- Step 5: Filter for the previous calendar year and group by month
SELECT
    customer_id,
    DATE_TRUNC('month', order_date) AS crossing_month
FROM first_crossing
WHERE EXTRACT(YEAR FROM order_date) = EXTRACT(YEAR FROM CURRENT_DATE) - 1;

```


Interview Follow-Up Questions & Answers
1. What if refunds should be subtracted instead of ignored?

Answer: You would join the orders table with the refunds table and calculate a "net_amount" (order_amount - COALESCE(refund_amount, 0)) before running the cumulative sum window function.

2. What if multiple orders happen on the same day?

Answer: To ensure the result is deterministic (consistent), you must refine the ORDER BY clause in your window functions to include a unique identifier, such as ORDER BY order_date, order_id.

3. How would you optimize this in Spark?

Repartitioning: Repartition the data by customer_id to minimize data shuffling across the cluster.

Range Partitioning: Use range partitioning to handle the sorting efficiently.

Caching: Use df.cache() on the initial filtered dataset to avoid re-reading from the source for each window operation.

4. How to make this incremental?

Answer: Store the "state" (the last known cumulative revenue) for each customer in a separate table. When new data arrives, process only the new records and add them to the stored state, rather than recalculating the entire history.

5. What if the threshold changes dynamically?

Answer: Avoid hardcoding values like 10000. Use a parameter or a configuration table so the threshold can be updated without changing the core query logic.

6. How to handle late-arriving data?

Answer: In a streaming environment, use watermarking to handle data that arrives late. In a batch environment, identify the specific customers affected by the late data and recompute the logic only for those IDs (Selective Re-processing).
